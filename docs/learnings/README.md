# Key Learnings

> Do not make the same mistake twice. First occurrence: learning. Second: systemic failure.

This directory captures mistakes, failures, and hard-won insights. Each entry represents something we now know that we didn't beforeâ€”and a system in place to prevent repetition.

---

## How to Add a Learning

When something goes wrong or you discover a blind spot:

1. Create a new file: `YYYY-MM-DD-brief-slug.md`
2. Use the template below
3. Ensure the prevention mechanism is concrete and verifiable
4. Link to any updated checklists, values, or agents

---

## Template

```markdown
# Learning: [Title]

**Date:** YYYY-MM-DD
**Category:** [operations/product/communication/strategy/process]
**Severity:** [minor/moderate/significant]

## What Happened
[Brief description of the mistake or failure]

## Why It Happened
[Root cause analysis - be honest]

## What We Learned
[The insight - what do we now know?]

## Prevention Mechanism
[Concrete system change - not "be more careful"]

- [ ] Checklist updated: [link]
- [ ] Value refined: [link]
- [ ] Agent updated: [link]
- [ ] Process changed: [description]

## Verification
How do we know the prevention works?
[Test, audit, or monitoring approach]

## Related
- Decision that led to this: [link]
- Similar past learnings: [links]
```
