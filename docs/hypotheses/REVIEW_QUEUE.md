## Queue Management

**When creating a hypothesis:**
1. Add to appropriate section based on review date
2. Link to both hypothesis doc and originating decision

**At session start:**
1. Check Overdue section first
2. Then This Week
3. Surface to user before proceeding with session

**After reviewing:**
1. Update hypothesis doc with outcome
2. Remove from queue OR move to Future if rescheduled
3. Create learning doc if invalidated
4. Update system components if warranted

**Monthly maintenance:**
1. Move items from Future to This Month as dates approach
2. Escalate any chronically deferred items
